UOL - New Checkout Project

Checkout for multiple stores, made with multiple steps, dynamic sales profiles to support A&B testing moving around fields on screen, their order and required validations.

Main challenges:
  • To implement dynamic multiple steps, based on Sales Profile .json config, set outside in a Mongo DB + Node.js under control of the business area.
  • Dynamic fields and validations, via Sales Profile either.
  • Build up a Domain Driven Validation, distributed in a single Java Script file able to be consumed by back-end apps and front apps. Rhino for Jaworks va Apps, and Front-End and with Node.js, natively.
  • Support Themes for different UOL's products(multi-store), without allow CSS customizations, customisation also coming from Sales Profile.
  • Support A/B testing set via Sales Profile, making the same product being able to have 2 or more different layouts, required fields and fields positions.
