UOL - Call Center: user history interface

UOL - Call Center: user history interface

A tracking page to full fill Contact Center attendants of information with UOL products usage.

This project took almost one semester to get done, but it's worth thetime expended to me and to UOL.

With these informations, attendants become able to make better decisions to cancel accounts when users ask for paybacks, claiming that they ever used any product or was signed up for any active selld force withou agreement.

A interesting part of this project, is that I made a simple framework very similar to Angular, even before I heard about it.

Build with LESS, Node.js, Grunt and Bower, provides controllers, models(json based) and views (with double keys assignment as Angular), this frame work is still in use and being evolved for others teams.

Yes, I'm a little bit proud of it...

Sign up Auto-complete Extension

UOL: signup auto-fill extension evolution

Evolution of an old UOL's firefox extension to auto-complete signup fields.

No one ask me to do it, instead, some people disagree.

But I saw a time savier on this, any test over UOL platform needs a 'fake user', and full fill the entire sign up form use to take up to 2 minutes.

So, I expended some of my free hours fixing it and bringing this extension back to life.

And add some new features:

· Generates just even CPF numbers, a requirement to bypass real validations of financial restrictions.

· Added a host file reader for Linux and Windows, so the user name automatically generated could contains prefixes that indicates different environment tests, described on the host file. A curiosity about it, the file reader lib was made for one of the ours leaders, that helped me a lot with this. Thanks Marco Tulio!

· Credit card generation evolution: Javascript version of Luh algoritym added to the core

· New sign up forms versions support added.
