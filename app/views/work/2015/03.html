SAC BOL: the UOL SAC's skin engine in action.

This is the first clone of UOL's SAC using an internal application called Chameleon, that allows non-dev users to customize their clones and keep them up to date with the main distribution.

Also the first Angular and single page app project in UOL's Chameleon, was a kind of pioneer work over a huge corporation as UOL, I'm glad to see it working.

Main challenges:

  • To configure out CORS using Nginx and to overcome a lot of cross domain issues using the SAC REST api, in other domain.
  • To avoid undesirable customizations in the clones, providing editable fields and controlled edition spots.
  • To provide a way to customize different states per pages.
  • Distribute app's controllers, directives, modules and providers over the same file and CDN to ingrease performance using controlled cache with md5 based renamed files.
  • To stipulate a consistent documentation and API to non-dev users
