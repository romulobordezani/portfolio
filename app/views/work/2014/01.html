UOL SAC: frontend architecture

New sac.uol.com.br, a responsive RESTful application using Angular, Bootstrap and ( Node.js + JAVA ).

Took me half year setting up the frontend infrastructure, learning new technologies and bringing them to a big tech company, where Development Operations are distributed in different teams.

With Internet Explorer 9 plain support, and keeping the old application blended, our team found a lot of challenges in this project.

Results:

  • High level code quality inspection using jsHint, that blocks deliveries with dirty code.
  • Continuous integration using Jenkins and under ITIL requirements.
  • Solid unit tests environment using Karma and Jasmine, that blocks deliveries with coverage under 80%.
  • Integrated with UOL's Custom Bootstrap.
  • New Angulartics metrics providers, customized to Adobe's Omniture UOL version.
  • Integrated validations in a single .js file, loaded in backend with Rhino and also consumed with Angular at the frontend.
  • Dependency distribution using Bower and npm.
  • Packages delivered using Apache Maven.
  • Automated static files distribution( CDN ) renaming files using MD5 to avoid cache on demand.
  • Support different appearances of the same application for different products, using Mustache.
  • Provides menu, banners and Frequent Answers and Questions customizations via Content Management System.
  • Go further then emulators and Virtual Machines, setting up an authenticated WIFI connection, redirected to Quality Assurance's Hosts, to test real devices.
